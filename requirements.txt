RIQUISITOS DEL PROYECTO
Impacto Global de la IA en el Empleo (2010-2025)

pyspark == 3.5.0
----------------------------------------------------------------
Interfaz Python para Apache Spark. Es el motor central del
pipeline. Los cuatro scripts de procesamiento (workload.py,
landing.py, curated.py, functional.py) son jobs de PySpark
ejecutados via spark-submit sobre YARN. Permite procesar el
dataset de forma distribuida, crear tablas Hive y escribir
resultados en formatos AVRO y Parquet.

pymongo == 4.6.1
----------------------------------------------------------------
Cliente oficial de Python para MongoDB. Se utiliza en los scripts
conexion_mongodb.py y migration_data_mongodb.py para exportar el
resultado de la capa Functional hacia la coleccion "gold" de
MongoDB (172.17.208.1:27017), que actua como capa de servicio
para Power BI.

pyarrow == 14.0.2
----------------------------------------------------------------
Biblioteca de serializacion columnar basada en Apache Arrow.
Requerida por PySpark para leer y escribir archivos en formato
Parquet (capas Curated y Functional). Sin esta dependencia las
escrituras Parquet fallan en entornos locales.

fastavro == 1.9.3
----------------------------------------------------------------
Implementacion rapida del formato AVRO en Python. Se utiliza para
validar y manipular el esquema ai_impact_jobs.avsc fuera del
contexto de Spark, y para pruebas de lectura de los archivos
generados en la capa Landing.

pandas == 2.1.4
----------------------------------------------------------------
Libreria de manipulacion de datos tabulares. Se emplea en tareas
de exploracion, validacion rapida de resultados y en el notebook
pyspark.ipynb del directorio datalakehouse/ para comparar
subconjuntos del dataset con DataFrames de Spark.

numpy == 1.26.4
----------------------------------------------------------------
Biblioteca de computo numerico. Dependencia transitiva de Pandas
y de las operaciones estadisticas realizadas durante el analisis
exploratorio del dataset.

python-dotenv == 1.0.1
----------------------------------------------------------------
Permite cargar variables de entorno desde el archivo .env en
tiempo de ejecucion. El proyecto usa este mecanismo para gestionar
parametros como la URI de conexion a MongoDB (MONGO_URI), rutas
HDFS y credenciales, evitando que se expongan en el codigo fuente
o en el repositorio publico de GitHub.

jupyter == 1.0.0
----------------------------------------------------------------
Entorno de notebooks interactivos. Requerido para ejecutar
pyspark.ipynb del directorio datalakehouse/, donde se realiza el
analisis exploratorio de los datos procesados y se documentan las
consultas analiticas sobre la capa Curated.

matplotlib == 3.8.3
----------------------------------------------------------------
Libreria de visualizacion. Se usa en los notebooks de exploracion
para generar graficos sobre distribucion de salarios, riesgo de
automatizacion por industria y adopcion de IA por anio.

seaborn == 0.13.2
----------------------------------------------------------------
Extension de matplotlib para graficos estadisticos. Complementa
el analisis exploratorio con mapas de calor, boxplots y graficos
de correlacion sobre las variables del dataset.

loguru == 0.7.2
----------------------------------------------------------------
Libreria de logging estructurado. Se usa en los scripts de
migracion para registrar eventos del pipeline con niveles y
formato legible, sin configuracion adicional.

tqdm == 4.66.2
----------------------------------------------------------------
Barras de progreso para scripts Python. Se aplica en los procesos
de carga y migracion para mostrar el avance en tiempo real.

pytest == 8.1.1
----------------------------------------------------------------
Framework de pruebas unitarias. Permite ejecutar pruebas de
validacion sobre las funciones auxiliares de los scripts del
pipeline, asegurando la integridad del codigo ante cambios.
